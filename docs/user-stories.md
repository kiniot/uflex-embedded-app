# uFlex Embedded User Stories

This document captures the embedded firmware user stories related to motion
capture, initial calibration, and telemetry generation for the uFlex IoT device.

| ID | Title | User Story | Acceptance Criteria | Epic |
| --- | --- | --- | --- | --- |
| TS40 | Embedded firmware for motion data capture | As a Developer, I want to implement the embedded data-capture logic from the IoT device sensors so the system can obtain initial joint-motion measurements during a rehabilitation session. | **Scenario 1: Successful sensor data capture**<br><br>**Given** the IoT device is powered on and the sensors are connected correctly,<br>**When** the firmware starts reading data,<br>**Then** the system captures motion values and prepares them for local processing.<br><br>**Scenario 2: Sensor unavailable**<br><br>**Given** the IoT device attempts to start capture,<br>**When** one of the sensors does not respond or is not connected correctly,<br>**Then** the system marks the device state as an error and avoids sending invalid data. | EP05 |
| TS41 | Initial calibration and validation of the IoT device | As a Developer, I want to implement an initial calibration and validation routine for the IoT device so captured measurements are consistent before starting a therapeutic session. | **Scenario 1: Calibration completed successfully**<br><br>**Given** the IoT device is in its initial configuration state,<br>**When** the firmware runs the calibration routine,<br>**Then** the system establishes baseline values for motion capture and allows the session to continue.<br><br>**Scenario 2: Calibration failed**<br><br>**Given** the IoT device attempts to calibrate,<br>**When** the captured values are outside the expected range,<br>**Then** the system keeps the error state and requires calibration to be repeated before telemetry capture. | EP05 |
| TS42 | Telemetry packet generation from embedded firmware | As a Developer, I want to generate telemetry packets from the embedded firmware so structured data can be sent to the uFlex Edge layer. | **Scenario 1: Telemetry packet generated successfully**<br><br>**Given** the IoT device captured valid motion data,<br>**When** the firmware builds the telemetry packet,<br>**Then** the system includes the device identifier, active context, estimated angle, proximal signal, and structured motion values.<br><br>**Scenario 2: Incomplete data**<br><br>**Given** the IoT device attempts to generate telemetry,<br>**When** required sensor or active-context data is missing,<br>**Then** the system discards or skips the packet and does not send invalid data to the Edge layer. | EP05 |
